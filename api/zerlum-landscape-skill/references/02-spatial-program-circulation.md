# 02 空间结构、功能布局与游线

## Purpose

建立清晰的空间骨架，让功能、人流、视线和体验顺序相互支持。

## Read First

- 主要人群、到达方式和高频目的。
- 必须设置的功能及其面积或容量依据。
- 安静与活跃、开放与私密、日常与活动的冲突。
- 建筑入口、城市界面、停车和后勤位置。
- 已确认的场地保留条件和设计方向。

## Design Actions

- 用一条主结构组织入口、核心空间和主要目的地。
- 通过次级路径、停留点和视线节点形成可选择的体验。
- 用相邻、分离、缓冲和共享四种关系组织功能。
- 检查步行、无障碍、骑行、车行、消防和服务流线是否冲突。
- 为入口、转折、核心、过渡和离场设置不同的空间节奏。

## Constraints

- 功能数量来自项目需求，不为填满总图而增加设施。
- 动线长度、宽度、坡度和容量没有依据时只作策略描述。
- 不以大尺度轴线替代真实的日常使用逻辑。

## Failure Signals

- 功能像图标一样散落，没有邻接逻辑。
- 所有路径同宽同级，没有主次和目的。
- 核心景观与建筑入口、运营或人群需求无关。
- 只考虑鸟瞰构图，不考虑人在地面的连续体验。

## Output Guidance

方案输出说明“结构是什么、为何成立、如何被使用、关键节点在哪里”。需要比较时，用同一评价维度对照两个或三个方向。
