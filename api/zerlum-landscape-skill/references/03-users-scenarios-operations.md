# 03 使用人群、场景与运营

## Purpose

让空间在不同人群、时间、季节和运营状态下都可理解、可使用、可维护。

## Read First

- 核心使用者、陪同者、工作人员和维护人员。
- 工作日、周末、活动日、早晚和季节差异。
- 停留、通行、社交、休憩、运动、儿童活动和服务需求。
- 开放时段、管理边界、商业经营和活动组织条件。

## Design Actions

- 为每类核心人群定义到达、使用、停留和离开的完整场景。
- 把高频日常使用与低频大型活动分开验证。
- 通过遮荫、座椅、饮水、卫生、导向和安全边界支持长时间使用。
- 将设备、保洁、垃圾、补给和维护路线纳入空间组织。
- 用可调整的场地而非过量固定设施适应变化。

## Constraints

- 不假定人群容量、运营时段或活动规模。
- 无障碍和安全要求需结合项目所在地现行规则复核。
- 不用“全龄”作为口号，必须说明不同人群如何被支持。

## Failure Signals

- 只描述拍照和视觉焦点，没有日常使用。
- 儿童、老人、照护者和维护人员被忽略。
- 活动布局与出入口、噪声或后勤冲突。

## Output Guidance

使用“人群 → 任务 → 时间 → 空间支持 → 运营条件”的结构。关键场景可转化为画布提示词中的人物活动和尺度参照。
