# 07 景观项目类型策略

## Purpose

识别不同项目类型的目标、矛盾和评价标准，避免用同一套景观形式处理所有场地。

## Read First

- 项目类型、设计阶段、客户目标和运营模式。
- 场地是否公共开放，是否有明确经营或管理主体。
- 核心人群、使用强度、开放时间和维护能力。

## Design Actions

- 居住景观：平衡归家、邻里、儿童、安静休憩、隐私和物业维护。
- 商业景观：组织可见性、到达、停留、店铺界面、活动和服务流线。
- 城市公园：优先公共性、可达性、日常活动、生态连续和耐久管理。
- 滨水景观：协调水位变化、安全、生态边缘、连续慢行和观水体验。
- 酒店度假：建立抵达、仪式、停留、私密、景观视线和运营服务顺序。
- 校园景观：支持学习、交往、运动、通行、成长变化和高频耐久。
- 文旅景观：用场地故事、路线、节点节奏、容量和运营形成体验，不依赖符号堆叠。

## Constraints

- 类型策略不能覆盖项目简报中的特殊要求。
- 商业与文旅不等于高饱和、强互动或持续活动。
- 居住与酒店不等于大门、水景和轴线的固定组合。

## Failure Signals

- 通过更换材质和植物名称伪装相同空间结构。
- 忽略运营、维护、公共性或服务流线的类型差异。
- 以案例印象替代场地证据。

## Output Guidance

只选择与当前项目相关的类型规则。混合项目需说明不同区域的主要目标和冲突解决方式。
