# 09 方案结构与交付

## Purpose

把项目依据、设计判断和方案成果组织成可汇报、可校对和可继续深化的交付文档。

## Read First

- 项目简报、场地资料和已确认设计方向。
- Agent 中明确形成的结论与待确认项。
- 画布中被采用的方案成果及其角色。
- 汇报对象、页数、设计阶段和输出格式。

## Design Actions

- 先定义整体叙事和页面角色，再写逐页内容。
- 从项目理解与场地问题进入设计命题和总体结构。
- 用功能、游线、节点、植物、材料和生态策略支撑概念。
- 为每页说明主要信息、版面层级、图像来源和是否需要生成图。
- 在末尾列出待复核项、下一步资料和专业协同需求。

## Constraints

- 只使用用户提交资料、已确认结论和画布方案成果作为项目依据。
- Skill 提供结构方法，不能被写成项目资料来源。
- 生成图不能作为测绘、工程、材料或规范证明。
- 不让整套方案每页都使用效果图。

## Failure Signals

- 目录固定套用，与项目类型和资料无关。
- 场地分析、概念和节点之间没有因果关系。
- 页面只有长文或大图，缺少明确页面任务。
- 未标注假设、来源和待复核内容。

## Output Guidance

大纲输出先写版式基调，再逐页列出页面角色、核心信息、主要视觉和图像使用。页面生成失败时保留已完成页面，支持单页重试。
