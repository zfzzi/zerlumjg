# 04 竖向、水与生态策略

## Purpose

让地形、水路径、海绵措施和生境逻辑成为空间设计的一部分，同时明确工程与专业复核边界。

## Read First

- 现状标高、坡向、低点、汇水边界和排水出口。
- 地下空间、建筑标高、道路和保留树根区。
- 水体类型、补水来源、水位变化和安全边界。
- 土壤、现状植被、生境连续性和已知污染风险。

## Design Actions

- 先建立雨水从哪里来、经过哪里、最终到哪里的路径。
- 用微地形、下凹绿地、植草沟、调蓄空间和溢流组织形成连续系统。
- 让台地、坡地、挡墙、台阶和无障碍路径共同解决高差。
- 保护有价值的现状土壤、树木和生境连接。
- 为枯水、暴雨、维护和儿童接近等状态设置安全策略。

## Constraints

- 无测绘、土壤、管线和计算资料时，不输出施工标高、容量或确定工程结论。
- 海绵、排水、水质、边坡和结构内容必须由相关专业复核。
- 不把水景视为任何项目的默认核心。

## Failure Signals

- 只画蓝色水面，没有水源、循环、溢流和维护逻辑。
- 地形造型与排水、无障碍或保留树冲突。
- 生态概念只有物种数量，没有生境结构和长期管理。

## Output Guidance

策略输出说明水路径、空间作用、运行状态和待复核输入。视觉提示词只表达可见的微地形、水边、湿生群落和材质状态，不伪造工程数据。
