# 08 景观视觉生成方法

## Purpose

把设计意图转换为可执行、可控制和可比较的图像或视频提示词，同时保护用户未要求改变的场地信息。

## Read First

- 任务模式：保结构优化、概念改造、局部深化、方向变体、季节时间变化或自由生成。
- 主图、场地底图和各参考图的显式角色。
- 用户要求保留与允许改变的内容。
- 镜头、空间尺度、人物活动和目标输出比例。

## Design Actions

- 保结构优化：保持视角、透视、地形、路径、建筑和主要空间边界，只调整景观层次与表达质量。
- 概念改造：说明允许改变的空间结构，并保留不可变条件。
- 局部深化：明确区域、对象、边界和与周边的衔接。
- 方向变体：保持评价条件一致，只改变一个或两个主导变量。
- 季节时间：保持方案结构一致，改变植物状态、天气、光线和使用情境。
- 视频：描述视线高度、路径、速度、转向、停留点、植被运动和人群连续性。

## Constraints

- 不默认蓝调夜景或湿润地面。
- 不无依据增加路径、水景、构筑物、地形或大规模人群。
- 植物需有成熟度、冠层、疏密和地域可信度。
- 材料需有尺度、接缝、边缘和真实反射。
- 人物用于说明尺度和活动，不遮挡核心空间。

## Failure Signals

- 参考图角色未区分，导致主体和风格混用。
- 只写氛围形容词，没有空间、植物和材料动作。
- 每次变体同时改变结构、镜头、季节和风格，无法比较。
- 建筑、道路或边界被模型随意重画。

## Output Guidance

用户要提示词时只输出提示词正文。按“保留条件、空间动作、植物、材料、人群、季节天气、镜头和质量限制”组织，但不写内部标签。
