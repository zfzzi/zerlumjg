# 01 场地分析、问题与机会

## Purpose

把场地资料转化为能驱动设计的约束、矛盾和机会，不把分析做成与方案脱节的信息罗列。

## Read First

- 区位、范围、边界和周边关系。
- 主要到达方向、出入口和现状流线。
- 地形高差、水系、汇水方向和低点。
- 保留建筑、树木、构筑物、视线和不利界面。
- 日照、风、噪声、热环境和季节变化的已知证据。

## Design Actions

- 将场地拆成必须保留、需要修复、可以强化和适合植入四类条件。
- 建立人流、车流、服务流和应急到达的初步关系。
- 识别视线焦点、遮挡、借景和需要缓冲的边界。
- 把高差与水路径转化为竖向和海绵策略线索。
- 每条分析结论都关联一个可能的设计响应。

## Constraints

- 无测绘或标高资料时，不给出确定坡度和高程。
- 无气候资料时，不断言主导风、积水或植物适生性。
- 航拍图、效果图和示意图不能替代法定边界或现状测量。

## Failure Signals

- 只复述文件名和图上对象。
- 机会点与后续空间策略没有对应关系。
- 把推测的地形、交通或水文条件当成事实。

## Output Guidance

优先使用“观察到的条件 → 影响 → 设计响应 → 待确认资料”的结构。场地资料不足时，列出最少且具体的补充项。
