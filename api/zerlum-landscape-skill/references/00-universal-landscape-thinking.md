# 00 通用景观设计思维

## Purpose

先建立设计问题、依据和评价标准，再提出空间动作。景观设计不是风格拼贴，而是对场地、人、时间、生态和建造条件的综合回应。

## Read First

- 用户希望解决什么问题，交付什么结果。
- 哪些内容来自项目简报和场地资料，哪些只是推断。
- 项目处于研究、概念、深化还是汇报阶段。
- 哪些条件不可改变，哪些内容允许探索。
- 谁使用场地，在什么时间、以什么方式使用。

## Design Actions

1. 用一句话定义核心设计命题。
2. 找出三至五个决定空间的关键条件。
3. 提出至少两个有实质差异的设计方向。
4. 把方向转译为空间结构、体验顺序和关键节点。
5. 检查人尺度、可达性、维护、建造和长期生长。

## Constraints

- 用户明确要求、项目简报和场地资料优先。
- 不把参考案例的形式直接复制到本项目。
- 不把未知条件写成事实。
- 不用效果图代替总图、竖向、排水、植物或工程判断。

## Failure Signals

- 先选风格，再解释场地。
- 只有“自然、高级、生态、未来”等形容词，没有空间动作。
- 所有项目都使用同一轴线、水景或大草坪。
- 忽略边界、出入口、服务流线和维护条件。

## Output Guidance

分析输出先写结论和依据；方向输出写命题、空间动作、体验和代价；评审输出按优先级说明问题、影响和修改建议。
