# 05 植物群落、季相与地域适应

## Purpose

用群落结构、空间功能和时间变化组织植物，而不是用随机树种或高密度花境装饰画面。

## Read First

- 项目所在地、气候与土壤资料是否明确。
- 保留树木、现状群落、视线和根系限制。
- 遮荫、屏障、导向、停留、雨水和生境需求。
- 维护能力、灌溉条件、过敏或安全风险。

## Design Actions

- 建立乔木、亚乔、灌木、草本和地被的层次关系。
- 先定义群落空间作用，再选择形态、季相和物种。
- 通过常绿与落叶、花果与叶色、疏林与密林形成时间变化。
- 预留植物成熟后的冠幅、遮挡、根系和维护空间。
- 将雨水花园、林下、滨水和活动场地使用相协调。

## Constraints

- 地域与气候资料不足时，只描述群落形态和性能，不给出确定树种表。
- 不以生成图中的植物作为适生性或苗木规格依据。
- 避免高维护、短期展示式配置成为默认答案。

## Failure Signals

- 所有树木大小一致、间距机械。
- 四季表达只靠改变叶色滤镜。
- 植物遮挡入口、导向、监护视线或安全边界。
- 用“乡土植物”替代具体的适应性判断。

## Output Guidance

方案文本使用“群落目标、层次、季相、空间作用、维护条件、待复核物种”。视觉提示词说明成熟度、疏密、冠层、林缘和季节，不堆叠物种名。
