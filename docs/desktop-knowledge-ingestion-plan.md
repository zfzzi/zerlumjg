## 2. 关键发现

- 资料结构完整，覆盖理论、光源、灯具、设计方法、设计手法、标准、计算、产品和项目实践。
- `01-05` 很适合作为 Zerlum 的基础照明知识、方案制作和效果图提示词上下文。
- `08_产品资料库` 和项目里的报价/灯具选型文件适合进入“灯具库”和“灯具报价”板块。
- `09_项目实践` 是高价值案例库，适合进入“数据库”和“文本制作”，也能给 AI 画布提供效果图参考。
- `06_标准与规范` 中存在旧版或版本不明标准，不能直接作为现行规范依据。
- 项目案例包含真实项目名、图纸、报价、效果图、CAD 和压缩包，必须设置隐私级别和权限。

## 3. 接入原则

不要把外部资料原样塞进 Agent。建议按“三层结构”接入：

1. `Agent 约束层`：当前 `agents/*/agent.md`，负责角色边界、路由、禁止行为、输出格式。
2. `知识资料层`：桌面资料库中的 Markdown、PDF、PPT、图片、报价表等，作为可检索内容。
3. `项目资产层`：真实项目图纸、报价、效果图、视频和压缩包，带权限、来源、隐私和版本控制。

Agent 只定义“怎么判断、怎么回答、怎么调用资料”；桌面文件是“被调用的知识和案例”，两者不要混在一个文件里。

## 6. 风险分级

| 风险 | 处理 |
| --- | --- |
| 旧标准或版本不明标准 | 进入 `standards-audit`，不得作为现行合规依据 |
| 项目隐私、客户名称、报价 | 标记 `private`，只给授权角色访问 |
| `.fas`、脚本、压缩包 | 隔离，不执行，不自动解包 |
| CAD/DWG 图纸 | 只建立文件索引和项目关联，后续用专业解析器处理 |
| 图片版权不明 | 只做内部参考，不进入正式交付素材 |
| 案例中的经验指标 | 可作为经验参考，必须回传规范 Agent 复核 |

## 10. 不建议做的事

- 不建议把桌面资料库直接复制到 `agents/`。
- 不建议用旧规范资料覆盖已建立的 `lighting-foundation` 标准索引。

- 不建议让 AI 自动读取或执行 `.py`、`.fas`、压缩包和 CAD 脚本。
