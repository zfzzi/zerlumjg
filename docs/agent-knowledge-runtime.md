# Zerlum 景观知识运行时

Zerlum 的生产知识上下文只有 `api/zerlum-landscape-skill/`。`SKILL.md` 负责总路由，`references/00-10` 覆盖场地分析、空间与游线、使用场景、竖向水生态、植物季相、材料细部、项目类型、视觉生成、文本交付和质量复核。

## 证据优先级

1. 用户明确要求；
2. 项目简报与场地资料；
3. 画布显式关系和已确认的 Agent 结论；
4. Landscape Skill 的方法框架。

Skill 不提供项目事实、现行规范合规证明、精确工程量或植物适生性结论。缺少依据时，输出必须标为假设或待复核项。

## 三个工作面

- 景观 Agent：理解任务、识别场地问题、比较方向并维护项目结论。
- 方案画布：以场地底图、现状照片和参考资料建立方案节点关系，生成方向、深化节点和漫游。
- 文本交付：只基于项目资料、已确认结论和画布成果生成大纲与页面，并区分事实、判断和待复核项。

## 生产边界

Vite 本地代理与生产 API 共用 `api/zerlum-landscape-skill.js`，部署配置只打包已审阅的 Landscape Skill Markdown。`knowledge/landscape-design-library/` 是可选的本地受控资料索引，不会自动进入模型上下文；资料须经过来源授权、隐私检查和专业复核后才可用于项目。
