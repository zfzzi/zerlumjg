# 景观工作台发布检查清单

## 产品范围

- [x] 工作台只保留景观 Agent、方案画布、文本交付三个入口。
- [x] 登录后直接进入工作台，无项目码、角色选择、成员管理或协作引导。
- [x] 项目上下文包含项目名、项目类型、地点、设计阶段和客户信息。
- [x] 深色与浅色主题均使用石墨灰与苔藓绿设计系统。

## Landscape Skill

- [x] 本地 Vite 与生产 API 共用版本化 Landscape Skill。
- [x] Skill 覆盖场地、功能游线、植物季相、材料构筑、生态水、制图表达和文本交付。
- [x] 提示词优先级为：用户明确要求 > 项目资料 > 画布显式关系 > 已确认设计结论 > 方法框架。
- [x] 旧照明 Skill、桌面照明知识库和检索脚本已移除。

## 方案画布

- [x] 空画布提供上传场地底图、上传现状照片、添加意向参考和创建方案图节点。
- [x] 支持保留结构、概念改造、局部深化、方向变体、季节时间和自由生成模式。
- [x] 连接关系区分场地底图、主场景、风格、材料、植物、首尾帧和视频参考。
- [x] 连接器具备稳定 48px 命中区、键盘打开、Esc 关闭和焦点恢复。
- [x] 节点错误显示具体信息并可重试当前节点。

## 文本交付

- [x] 四阶段为资料确认、大纲生成、页面生成、校对与导出。
- [x] 大纲包含场地问题、总体空间、功能游线、植物季相、材料构筑和待复核项。
- [x] 页面生成携带项目身份并约束事实、判断与待复核项边界。
- [x] 单页失败可独立重试，已完成页面保持不变。
