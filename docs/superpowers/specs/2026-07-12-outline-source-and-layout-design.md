# Zerlum Outline 专职大纲与排版风格设计

## 目标

将 Zerlum Outline 收窄为文本交付工作流中的专职编排 Agent。它只根据产品显式提交给它的项目资料生成方案大纲，并为整套文本交付确定统一的排版样式风格。它不生成最终页面，不参与普通 Agent 对话，也不调用联网检索或未传入的数据源。

## 输入边界

Outline 可以读取以下显式来源：

1. 文本交付区上传的文件、图片及可提取文本；
2. 用户在文本交付区输入的本次要求；
3. 左侧项目简报中的项目名称、类型、地点、阶段、客户、目标、使用人群、场地范围和已知限制；
4. Zerlum Agent 中状态正常的用户与助手对话，以最近的有效内容为主；
5. 方案画布中已生成的有效图片及其标签；
6. 文本交付区当前已有大纲，用于用户要求的局部优化或重新编排。

Outline 不读取数据库、平台导航状态、隐藏页面内容、agent.md 或联网检索结果。Landscape Skill 只能提供景观专业方法与质量约束，不能作为项目事实来源。

## API 隔离

Outline 使用独立的 OpenAI 兼容通道配置：

- `OPENAI_OUTLINE_API_KEY`
- `OPENAI_OUTLINE_BASE_URL`
- `OPENAI_OUTLINE_MODEL`

生产环境将专用密钥与地址绑定到用户指定的 qweapi 通道。为保持向后兼容，专用变量缺失时才回退到 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 和 `OPENAI_AGENT_MODEL`。Agent、提示词、图片和文档页面生成通道不受影响。

## 输出契约

Outline 输出结构化 Markdown，且只包含排版规范与分页大纲。

### 1. 整套排版风格

输出开头必须包含 `【整套排版风格】`，并明确：

- 16:9 横屏比例；
- 风格名称与选择依据；
- 主色、辅助色、强调色及使用比例；
- 中文与西文字体方向、标题与正文字级关系；
- 网格、页边距、留白和对齐规则；
- 图片裁切、蒙版、边框和色调处理；
- 图表、图标、分析图和标注语言；
- 封面、章节页、分析页、策略页和效果图页的统一关系。

该风格块位于第一页标题之前。现有分页解析会把页前公共内容传给每一页，因此后续页面生成 Agent 能继承同一套风格。

### 2. 分页大纲

每页使用明确的 `第 N 页` 标题，并逐页包含：

- 页面标题与页面类型；
- 本页目的和关键信息；
- 主要视觉元素；
- 内容位置、网格与图文层级；
- 是否使用画布生成图及使用方式；
- 资料依据、设计判断和待复核项。

效果图只用于封面、方向表达、重点节点、关键体验或前后对比等必要页面。其他页面优先使用场地分析、空间结构、功能、游线、植物、材料、节点、生态水或运营时间线等专业表达。

## 数据流

1. 文本交付界面汇总六类显式输入并提交 `view: "text"`、`agentTask: "outline"`。
2. `/api/zerlum-agent` 识别 Outline 任务，优先读取 Outline 专用密钥、地址和模型。
3. Outline 返回非流式 Chat Completions 结果；代理沿用现有 SSE 包装方式更新界面。
4. 前端保存完整结构化 Markdown，并按 `第 N 页` 拆分。
5. 页前的排版风格块被带入每个分页提示词，页面生成 Agent 据此保持整套视觉一致性。

## 空输入与错误处理

- 只有用户指令、但没有项目简报有效字段、上传资料、有效 Agent 对话、画布成果或已有大纲时，不调用 Outline API，提示先提交资料。
- qweapi 认证、网络或模型错误沿用现有可读错误转换，不泄露密钥或上游响应细节。
- Outline 返回空内容时保留现有大纲，不生成虚构页面。
- 资料冲突时在大纲中标记待复核，不自行选择为项目事实。

## 测试与验收

- 失败测试先证明 Outline 尚未优先读取专用 API 密钥和地址。
- 路由测试验证 Outline 专用配置优先、公共配置仅为回退，其他 Agent 路由不变。
- 前端测试验证项目简报、上传资料、Agent 对话、画布成果和已有大纲都会进入 Outline 上下文。
- 输出约束测试验证存在 `【整套排版风格】`、统一视觉规则和逐页排版字段。
- 空输入测试验证不会误把 Landscape Skill 或用户空泛指令当成项目资料。
- 全量测试、TypeScript 构建和 Vercel 函数打包必须通过。
- 生产部署后验证 Outline 路由使用专用 qweapi 环境变量，网站可生成包含统一排版风格的分页大纲，浏览器控制台无新增错误。
